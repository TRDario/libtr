Detailed Description

GPU index buffer class.

An OpenGL context must be open to instantiate and use objects of this type.

◆ mapNew()

IndexBufferMap tr::IndexBuffer::mapNew ( std::size_t size)
export

Maps the buffer, invalidating the previous contents in the process and resetting its size.

If size is greater than the capacity of the buffer, a reallocation will be done. This voids any previous bindings of the buffer to the context, and so it must be rebound. mapRegion() will never cause a reallocation, so may be used in cases where that's a requirement.

Exceptions
GLBufferBadAllocIf a reallocation was triggered and reallocating the buffer failed.
GLMapBadAllocIf mapping the buffer failed.
Parameters
sizeThe new size of the buffer in indices. Must be greater than 0, otherwise a failed assertion may be triggered.
Returns
A map object.

IndexBufferMap tr::IndexBuffer::mapRegion ( std::size_t offset,
std::size_t size )
export

Maps a region of the buffer, invalidating the previous contents in the process.

Unlike mapNew(), a call to this function will never cause a reallocation, but an assertion may fail if an out-of-bounds map is requested.

Exceptions
GLMapBadAllocIf mapping the buffer failed.
Parameters
offsetThe starting offset within the buffer in indices.
sizeThe size of the mapped region in indices. Must be greater than 0, and offset + data.size() <= capacity() must hold true, otherwise a failed assertion may be triggered.
Returns
A map object.
