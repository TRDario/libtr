◆ mapNew()

GraphicsBufferMap tr::VertexBuffer::mapNew ( std::size_t size)

Maps the buffer, invalidating the previous contents in the process and resetting its size.

Note
If size is greater than the capacity of the buffer, a reallocation will be done. This voids any previous bindings of the buffer to the context, and so it must be rebound. mapRegion() will never cause a reallocation, so may be used in cases where that's a requirement.
Exception Safety

Weak exception guarantee, if mapping the buffer after reallocation fails, the reallocation is not reverted.

Exceptions
GraphicsBufferBadAllocIf a reallocation was triggered and reallocating the buffer fails.
GraphicsBufferMapBadAllocIf mapping the buffer fails.
Parameters
[in]size

The new size of the buffer in bytes.

Precondition
size must be greater than 0.
Returns
A map object.
